# 📘 Cahier des charges – Plateforme CodeQuest 🎮

## 1. Contexte et problématique

Dans l'enseignement des sciences informatiques au secondaire qualifiant, les élèves rencontrent souvent des difficultés dans la compréhension des concepts d'algorithmique et de programmation, notamment en raison de :

* L'abstraction des notions
* Le manque de pratique progressive
* L'anxiété liée à la programmation
* L'absence de motivation

Dans ce contexte, le projet **CodeQuest** propose une plateforme numérique gamifiée visant à faciliter l'apprentissage progressif de l'algorithmique et du langage Python.

---

## 2. Objectifs du projet

### 🎯 Objectif général

Développer une plateforme interactive permettant aux élèves d'apprendre l'algorithmique et Python de manière progressive, motivante et autonome.

### 🎯 Objectifs spécifiques

* Renforcer la compréhension des bases de l'algorithmique
* Assurer une transition fluide vers Python
* Encourager l'apprentissage actif à travers des exercices progressifs
* Réduire l'anxiété grâce à un environnement ludique
* Suivre la progression des apprenants

---

## 3. Description générale de la plateforme

La plateforme **CodeQuest** est une application web éducative intégrant :

* Un système d'authentification des élèves
* Un parcours pédagogique structuré en étapes
* Des activités variées (QCM, exercices, mini-projets)
* Un système de gamification (points, badges)
* Un tableau de bord individuel

---

## 4. Fonctionnalités principales

### 4.1 Authentification

* Inscription / Connexion de l'élève
* Identification par nom ou identifiant
* Accès à un espace personnel

---

### 4.2 Tableau de bord (Dashboard)

Chaque élève dispose d'un dashboard affichant :

* Nombre total de points
* Badges obtenus
* Progression (%)
* Étapes complétées
* Historique des activités

---

### 4.3 Gamification

* Attribution de points pour chaque activité réussie
* Système de badges par niveau ou compétence
* Barre de progression
* Motivation par feedback immédiat

---

## 5. Parcours pédagogique détaillé

Le parcours est structuré en **4 grandes étapes** :

---

## 🟢 Étape 1 : Évaluation des prérequis en algorithmique

### 🔹 Activité 1 : QCM

* Objectif : vérifier la compréhension de base
* Exemple :

  * "Pour afficher une valeur, on utilise :
    a) Lire
    b) Écrire
    c) Si"

---

### 🔹 Activité 2 : Exercices simples (2 exercices)

Chaque exercice comprend :

* Énoncé
* Solution fournie
* Zone de saisie de l'algorithme

#### ⚙️ Fonctionnement :

* Temps limité : **10 minutes**
* L'élève écrit sa solution
* Comparaison automatique avec la correction
* Bouton **"Afficher la correction"**

---

### 🔹 Activité 3 : Structures conditionnelles simples (2 exercices)

* Utilisation de "Si... Alors..."
* Exercices guidés avec validation

---

### 🔹 Activité 4 : Structures conditionnelles imbriquées (2 exercices)

* Introduction aux conditions imbriquées
* Exercices progressifs

---

### 🔹 Activité 5 : Exercices complexes (2 exercices)

* Mobilisation de toutes les notions
* Résolution de problèmes plus réalistes

---

## 🔵 Étape 2 : Transition vers Python

### 🔹 Introduction

* Comparaison entre algorithmique et Python
* Tableau de correspondance (Écrire → print, Si → if…)
* Résumé synthétique

---

### 🔹 QCM Python

* Vérification de compréhension
* Questions sur syntaxe Python

---

## 🟣 Étape 3 : Pratique en Python

### 🔹 Exercices simples (2 exercices)

* Utilisation de print, variables

### 🔹 Conditions simples (2 exercices)

* if / else

### 🔹 Conditions imbriquées (2 exercices)

* logique avancée

---

## 🔴 Étape 4 : Mini-projets guidés

### 🔹 Projet 1 : Calculatrice simple

* Interface utilisateur simple
* Opérations de base

### 🔹 Projet 2 : Projet guidé

* Exemple : gestion de notes / jeu simple

#### ⚙️ Particularité :

* Intégration d'un **éditeur de code Python interactif**
* Exécution directe du code
* Feedback immédiat

---

## 6. Contraintes techniques

* Application web (HTML, CSS, JavaScript)
* Intégration d'un moteur Python (ex : Pyodide)
* Stockage local ou base de données simple
* Interface simple et intuitive

---

## 7. Critères de réussite

* Amélioration des résultats des élèves
* Engagement et motivation accrus
* Progression visible dans le dashboard
* Capacité des élèves à réaliser un mini-projet

---

## 8. Perspectives d'évolution

* Ajout de comptes enseignants
* Suivi global de classe
* Ajout de nouveaux niveaux
* Intégration de Scratch ou AlgoBox

---

## 🎯 Conclusion

Le projet **CodeQuest** constitue une solution innovante intégrant pédagogie active et technologies numériques, favorisant un apprentissage progressif, autonome et motivant de la programmation.
